## 二、值得借鉴的产品决策

### 1. "可靠"比"炫酷"先走一步

Claude 2.1做的最重要的事不是加功能，而是把幻觉率降50%。这为后面Agent能力打基础——你不信任一个工具，就不会让它连续跑7小时。

**产品启示**：做Agentic工具，先把交付可靠性做到极致，再谈功能扩展。

### 2. 用"分层定价"做能力验证和商业化

Opus/Sonnet/Haiku三层，本质是在测试市场对不同能力等级的付费意愿。Sonnet 3.5以1/5价格逼近Opus水平时，说明"够用的Agent能力"才是最大市场。

**产品启示**：Agentic工具也要分层——轻量版走量，专业版做深度，不要一上来就All-in旗舰。

### 3. 先做"单点深度"再做"多点并行"

Claude 4先解决"7小时连续运行不衰减"这个单点，到4.8才做Dynamic Workflows的并行调度。

**产品启示**：垂直领域Agentic工具，先把一个场景的端到端交付做深做透，再考虑横向扩展。

### 4. Skills = 把专业知识"打包"给Agent

Agent Skills解决了"LLM不擅长确定性任务"的问题——写Excel公式、填PDF表单这些，交给可执行代码，而不是让模型硬猜。

**产品启示**：垂直Agentic工具的竞争力，在于把领域知识+确定性工具封装成可复用的"技能包"，而不是光靠模型能力硬撑。

## 三、我的分析

纯聊天或基础AI Coding工具将逐渐被能力更强的Agentic产品替代，市场最终会收敛到几家头部产品，简化用户操作，无需管理多个产品。

**核心判断**：
- Agentic工具是大势所趋，用户需要的是能够真正干活且交付质量高的Agentic工具产品
- 重点关注垂直领域+Agentic工具的落地应用机会
- 大厂做通用Agent平台，垂直领域的深度know-how、行业数据壁垒、合规要求，是产品切入的护城河
- "大厂修路，你在路上跑专车"的思路

## 四、垂直领域+Agentic工具落地机会

| 垂直领域 | Agentic切入点 | 为什么现在是窗口期 |
|---------|-------------|----------------|
| **法律** | 合同审阅Agent：自动标注风险条款、生成修改建议、对比版本差异 | Claude的可靠性提升（幻觉率持续下降）让法律场景首次可用 |
| **财税** | 报税/审计Agent：自动抓取票据、校验合规性、生成申报表 | 确定性计算+规则判断，正好是Skills机制的甜区 |
| **医疗** | 病历整理+辅助诊断Agent：结构化提取、知识检索、报告生成 | 长上下文(1M)让完整病历终于能一次性处理 |
| **制造业** | 产线质检/排产Agent：图像检测+排程优化+异常报警 | 多模态+Computer Use让"看图干活"成为可能 |
| **金融研究** | 研报生成Agent：数据采集→分析→成文→可视化一气呵成 | Dynamic Workflows的并行能力恰好适合多源数据处理 |
